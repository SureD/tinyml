TinyInfer CPU Backend Design

Design doc for the implemented CPU f32 correctness backend: memory boundary, operator contracts, KV cache behavior, move safety, and verification coverage.

StatusImplemented as the f32 correctness backend.
Parent designTinyInfer Backend Design
Backend roleNumerical oracle for later Metal kernels.
Main constraintNo hidden heap allocation inside CPU ops.

1. Scope

The CPU backend is intentionally simple: single-threaded scalar kernels, strict shape checks, no hidden heap allocation in operator hot paths, and no dtype conversion. It is not the final performance target. It exists to make correctness observable before Metal kernels and lower-precision paths are added. 

MemoryArena -> native CPU allocation
TensorView  -> byte offset + shape + strides
CPUBackend  -> validate view, compute pointer, run op

TensorView owns no memory and never interprets backend storage. CPUBackend is the only code that turns a MemoryArena native handle into a host pointer.

2. Common Contract

  • Every math op requires f32 contiguous tensors owned by the same CPU backend.
  • Bad shapes, foreign arenas, undefined views, wrong devices, and non-contiguous views return invalid_argument.
  • Unsupported dtypes return unimplemented.
  • Output views must be caller-provided. CPU ops do not allocate temporary tensors.
  • Copies are raw byte copies and do not perform dtype conversion.

3. Operator Contracts

3.1 Matmul

matmul_out(out, x, w)

x:   [M, K]
w:   [N, K]
out: [M, N] or any contiguous shape with first dim M and M*N elements

out[m,n] = sum_k x[m,k] * w[n,k]

The [N,K] weight layout matches PyTorch and Hugging Face linear weights. This avoids a loader-side transpose during early bring-up and keeps checkpoint parity easier to reason about.

3.2 RMSNorm

rms_norm_out(out, x, weight, eps)

x:      [T, H]
weight: [H]
out:    [T, H]

For each row, compute f32 mean(x^2), then write x * rsqrt(mean + eps) * weight. In-place out == x is valid because the row reduction completes before writing.

3.3 RoPE

rope_inplace(q, k, start_pos, theta)

q: [T, H, D]
k: [T, KVH, D]
D must be even

RoPE uses the TinyLlama/LLaMA half-split rotation: 

x0 = x[i]
x1 = x[i + D/2]
angle = position / theta^(2*i/D)
y[i]       = x0*cos(angle) - x1*sin(angle)
y[i + D/2] = x1*cos(angle) + x0*sin(angle)

theta is explicit in the backend API so the kernel does not hard-code model config.

3.4 Attention

attention_out(out, q, k, v, k_cache, v_cache, start_pos, kv_len)

q:       [T, H, D]
k, v:    [T, KVH, D]
k_cache: [KVH, S, D]
v_cache: [KVH, S, D]
out:     [T, H*D] or [T, H, D]

The engine passes per-layer cache views. The op first writes current k/v into cache positions [start_pos, start_pos + T). It then computes grouped-query causal attention over keys [0, kv_len), where query head h maps to KV head h / (H / KVH).

The CPU implementation recomputes scores during softmax instead of allocating a score buffer. That is slower but keeps ownership explicit and makes the baseline easy to debug.

3.5 SwiGLU

swiglu_out(out, gate, up)

out[i] = silu(gate[i]) * up[i]
silu(x) = x / (1 + exp(-x))

out, gate, and up must have exactly the same shape.

3.6 Argmax

argmax(out_token, logits) accepts [V] or [1,V] logits and returns the first maximum index. Tie behavior is deterministic.

4. Engine Integration

The KV arena is stored as [layer, kv_head, position, head_dim]. LlamaInferEngine derives a per-layer [kv_head, position, head_dim] view before calling attention_out, keeping layer indexing out of the backend kernel.

LlamaInferEngine has custom move construction and move assignment. Moving an engine also rebinds all internal TensorView::arena pointers to the destination arenas. Without this, views would point at moved-from arenas and fail backend validation.

5. Non-Goals

  • No SIMD or threading yet.
  • No f16 or bf16 arithmetic yet.
  • No Metal implementation in this step.
  • No hidden temporary tensor allocation inside CPU ops.
  • No generic graph or operator registry.

6. Verification

  • CPU arena allocation, copy, and argmax.
  • Matmul numerical output and shape failures.
  • RMSNorm and SwiGLU numerical output.
  • RoPE half-split rotation.
  • Grouped-query causal attention, including KV cache writes.
  • Engine move safety through the fake backend flow test.
